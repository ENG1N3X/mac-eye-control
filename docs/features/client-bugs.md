## ✅ Issue 1: Объяснение как работает RESET HEAD POSE [DONE]

**Проблема:** Пользователи не понимают, что делает кнопка/клавиша сброса позы головы.

**Текущее поведение:**
- Клавиша `C` вызывает `head_pose.recalibrate()`, которая сохраняет текущее положение головы как "нулевое"
- Это нужно для корректного вычисления питча при прокрутке

**План решения:**
1. **Документация:**
   - Добавить подробный комментарий в `main.py` с объяснением, зачем нужен сброс
   - При запуске приложения вывести в консоль объяснение всех клавиш

2. **UI улучшения:**
   - При нажатии `C` показать всплывающее сообщение на видео на 2 сек: "Head pose reset - look straight ahead"
   - В левом верхнем углу видео вывести текущее положение головы (pitch, yaw, roll)

3. **Интеграция в код:**
   - Файл: `src/control/head_controller.py` или `main.py`
   - Логика: когда `recalibrate()` вызывается, установить флаг и показать визуальный индикатор

## ✅ Issue 2: Если с помощью мыши или тачпада двигаешь курсор, отключать перемещение курсора глазами [DONE]

**Проблема:** Когда пользователь вручную двигает курсор мышью/тачпадом, газ-контроль продолжает его перемещать, создавая конфликт.

**План решения:**
1. **Отслеживание движения мыши:**
   - Добавить модуль `src/control/mouse_monitor.py` с классом `MouseMonitor`
   - Использовать `pynput.mouse.Listener` для отслеживания движения и кликов мыши
   - При детектировании движения мыши: установить флаг `manual_mouse_active = True`

2. **Таймер отключения:**
   - При движении мыши запустить таймер (0.5-1 сек)
   - После истечения таймера и отсутствия новых движений вернуть флаг в `False`
   - `CursorController.move()` проверяет флаг перед перемещением

3. **Интеграция:**
   - В `main.py`: инициализировать `MouseMonitor`
   - Перед `cursor.move()`: проверить `if not mouse_monitor.is_manual_active(): cursor.move(...)`

4. **Параметры в конфиг:**
   - `manual_mouse_timeout_sec`: 0.5 (сек задержки перед возобновлением газ-контроля)

## ✅ Issue 3: Помехи курсора - постоянно двигается [DONE]

**Проблема:** Курсор дрожит, постоянно прыгает, невозможно навести на маленькую цель.

**Возможные причины:**
- Недостаточное сглаживание (smoothing_window слишком мал)
- Погрешности калибровки
- Высокие частоты в сигнале ириса

**План решения:**
1. **Увеличить сглаживание:**
   - По умолчанию `smoothing_window = 10` → увеличить до `15-20`
   - Использовать экспоненциальное скользящее среднее (EMA) вместо простого буфера
   - Коэффициент: `alpha = 2 / (window_size + 1)`

2. **Добавить deadzone:**
   - Если движение курсора `< 2px` за кадр → не двигать
   - Это подавит микротремор

3. **Улучшить калибровку:**
   - Увеличить `calibration_dwell_sec` с 1.0 до 1.5 сек (дольше собирать данные)
   - Добавить фильтр выбросов при сборе калибровочных данных (отбросить 10% крайних значений)

4. **Интеграция:**
   - Файл: `src/control/cursor.py`
   - Добавить методы: `_apply_deadzone()`, `_apply_exponential_smoothing()`
   - Параметры в конфиг: `cursor_deadzone_px: 2`, `smoothing_alpha: 0.15`

## ✅ Issue 4: Калибровка должна срабатывать когда пользователь точно посмотрел на точку, а не по таймауту [DONE]

**Проблема:** Текущий механизм ждёт фиксированное время (`calibration_dwell_sec`), что приводит к неточной калибровке если пользователь отвлёкся.

**Текущий механизм:** статический таймер

**План решения:**
1. **Детектирование фиксации взгляда (Gaze Fixation Detection):**
   - Добавить класс `GazeFixationDetector` в `src/tracking/`
   - Логика: если стандартное отклонение ириса за последние N кадров `< threshold` → фиксация обнаружена
   - Требуемое время фиксации: 0.3-0.5 сек (вместо 1.0 сек по таймауту)

2. **Параметры:**
   - `fixation_std_threshold`: 15 (пиксели в окне иризации)
   - `fixation_duration_sec`: 0.3 (минимальная длительность)
   - `fixation_window_frames`: 15 кадров (для сбора статистики)

3. **Интеграция в `CalibrationSession`:**
   - Вместо `time.sleep(calibration_dwell_sec)` использовать цикл:
     ```python
     while not fixation_detector.is_fixated():
         # читать кадры и обновлять детектор
         pass
     # собрать усреднённые значения
     ```

4. **UI:**
   - Показать прогресс-бар когда фиксация обнаружена
   - При отсутствии фиксации: текст "Hold gaze on point..."

## Issue 5: Перекалибровка во время работы приложения (клавиша R)

**Проблема:** Нажатие `R` во время работы должно запустить экран калибровки и пересоздать `calibration.json`, но это может не работать корректно.

**План решения:**
1. **Реализовать hotkey обработчик для R:**
   - Добавить в `main.py` проверку на нажатие `R` в главном цикле
   - Когда `R` нажата: установить флаг `recalibration_requested = True`

2. **Безопасное переключение в режим калибровки:**
   - Остановить основной control loop (курсор, клик, скролл не работают)
   - Переинициализировать `CalibrationSession` и запустить `.run()`
   - После завершения: перезагрузить `GazeMapper` из нового `calibration.json`
   - Возобновить control loop

3. **UI индикатор:**
   - Во время калибровки: заморозить видео или показать тёмный оверлей
   - После завершения: подтверждение "Calibration saved. Resuming gaze control."

4. **Обработка ошибок:**
   - Если калибровка не завершена (пользователь нажал `Q` или произошла ошибка) → вернуться к предыдущему `calibration.json`
   - Не потерять работающую калибровку

5. **Интеграция:**
   - Файл: `main.py`, добавить состояние `app_state: Enum = {CONTROL, CALIBRATION}`
   - При `app_state == CALIBRATION`: не обновлять курсор, запустить UI калибровки

## Issue 6: Не работает когда смотришь вверх (только 8/10 экрана работает)

**Проблема:** Верхние 2/10 экрана недоступны для газ-контроля; курсор не может туда попасть.

**Возможные причины:**
- MediaPipe не корректно отслеживает глаза при экстремальных углах (pitch близко к ±45°)
- Камера плохо видит глаза при сильном наклоне головы
- Калибровка не полностью охватывает весь диапазон углов
- Маппинг (GazeMapper) экстраполирует неправильно вне обучающего набора

**План решения:**
1. **Расширить диапазон калибровки:**
   - Вместо 3x3 сетки (9 точек) → использовать 5x5 сетку (25 точек) с фокусом на верхнюю область
   - Или минимум добавить 3 точки в верхний ряд

2. **Улучшить отслеживание:**
   - Проверить `min_detection_confidence` и `min_tracking_confidence` в MediaPipe
   - Может потребоваться снизить до 0.5-0.6 для лучше работы при экстремальных углах

3. **Оптимизировать маппинг:**
   - Использовать более гибкий алгоритм: вместо полиномиальной регрессии 2-го порядка → RBF (радиальная базисная функция) или нейросеть
   - Или добавить локальную регуляризацию для краев экрана

4. **Тестирование:**
   - Проверить логи калибровки: какие `pitch` углы были зафиксированы во время калибровки
   - Убедиться что максимальный pitch >= 35° (для верхней трети экрана)

5. **Интеграция:**
   - Файл: `src/calibration/mapping.py`
   - Опция: `mapping_method: "polynomial" | "rbf"` в конфиге
   - Параметр: `calibration_grid_size: 3 | 5` (по умолчанию 5)

6. **Workaround на короткий срок:**
   - Добавить клипирование: если predicted `screen_y < 50px` → установить `screen_y = 50px`
   - Это даст доступ к верхней части, но не идеально
